# 任务管理系统部署文档

## 1. 部署目标

本文档用于将任务管理系统部署到 Ubuntu 或 WSL Ubuntu 环境，并通过 Nginx 对外提供访问。

当前推荐部署方式：

- 前端：`web/dist` 静态文件由 Nginx 托管
- 后端：Node.js 运行 `server/dist/index.js`
- 进程守护：PM2
- 反向代理：Nginx
- 访问端口：`8888`

## 2. 服务器要求

推荐环境：

| 软件 | 推荐版本 |
| --- | --- |
| Ubuntu | 22.04 或以上 |
| Node.js | 20 或以上 |
| npm | 10 或以上 |
| Nginx | 1.24 或以上 |
| PM2 | 最新稳定版 |

检查命令：

```bash
node -v
npm -v
nginx -v
```

## 3. 项目目录

推荐部署目录：


主要目录说明：

```text
TimeManageSystem
├── server              后端服务
├── web                 前端工程
```

默认数据库文件是 `server/server-data.db`。如需放到其他磁盘或目录，可在 `.env` 中配置 `DB_PATH`。

## 6. 配置环境变量

创建 `.env`：

```bash
nano /home/lizb/TimeManageSystem/.env
```

生产环境示例：

```env
PORT=4000
JWT_SECRET=change-this-secret
DB_PATH=/home/lizb/TimeManageSystem/server/server-data.db

DINGTALK_APP_KEY=你的钉钉 AppKey
DINGTALK_APP_SECRET=你的钉钉 AppSecret
DINGTALK_WORKBOOK_ID=你的钉钉表格 WorkbookId
DINGTALK_OPERATOR_ID=有表格权限的用户 ID
DINGTALK_API_BASE_URL=https://api.dingtalk.com

PUBLIC_BASE_URL=http://172.30.0.50:8888
FRONTEND_BASE_URL=http://172.30.0.50:8888
```

说明：

| 配置项 | 说明 |
| --- | --- |
| PORT | 后端监听端口，默认 4000 |
| JWT_SECRET | 登录 token 加密密钥，生产环境必须改成复杂字符串 |
| DB_PATH | SQLite 数据库文件路径，不配置时默认使用 `server/server-data.db` |
| DINGTALK_APP_KEY | 钉钉开放平台 Client ID |
| DINGTALK_APP_SECRET | 钉钉开放平台 Client Secret |
| DINGTALK_WORKBOOK_ID | 钉钉在线表格工作簿 ID |
| DINGTALK_OPERATOR_ID | 有表格读写权限的钉钉用户 ID |
| PUBLIC_BASE_URL | 外部访问后端和 OAuth 回调使用的公网或内网地址 |
| FRONTEND_BASE_URL | 前端访问地址 |

如果访问地址或端口变化，必须同步修改：

- `.env` 中的 `PUBLIC_BASE_URL`
- `.env` 中的 `FRONTEND_BASE_URL`
- 钉钉开放平台回调地址
- Nginx 监听端口

## 9. 配置 Nginx

创建配置文件：

```bash
sudo nano /etc/nginx/sites-available/task-manage
```

写入以下配置：

```nginx
server {
    listen 8888;
    server_name _;

    root /home/lizb/TimeManageSystem/web/dist;
    index index.html;

    location /api/ {
        proxy_pass http://127.0.0.1:4000/api/;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    location /health {
        proxy_pass http://127.0.0.1:4000/health;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    location / {
        try_files $uri $uri/ /index.html;
    }
}
```

启用站点：

```bash
sudo ln -s /etc/nginx/sites-available/task-manage /etc/nginx/sites-enabled/task-manage
```

如果默认站点占用配置，可以删除默认站点：

```bash
sudo rm -f /etc/nginx/sites-enabled/default
```

检查配置：

```bash
sudo nginx -t
```

重载 Nginx：

```bash
sudo systemctl reload nginx
```

## 11. 钉钉开放平台配置

进入钉钉开放平台，找到企业内部应用。

需要确认：

### 11.1 应用凭证

获取：

- Client ID / AppKey
- Client Secret / AppSecret
- CorpId

### 11.2 权限申请

至少需要：

- 读取用户信息权限
- 通讯录读取权限
- 在线表格读取权限
- 在线表格写入权限

如果登录时报：

```text
Contact.User.Read
```

说明缺少读取用户信息权限，需要在钉钉开放平台申请并发布。

### 11.3 回调地址

安全设置中配置：

```text
http://172.30.0.50:8888/api/auth/oauth/dingtalk/callback
```

端内免登地址可配置：

```text
http://172.30.0.50:8888
```

如果本地开发也要登录，可额外配置：

```text
http://localhost:4000/api/auth/oauth/dingtalk/callback
http://localhost:5173
```

注意：回调地址必须与系统实际生成的地址完全一致，协议、IP、端口、路径都不能错。

## 13. Windows 访问 WSL 部署

如果系统部署在 WSL，Windows 本机一般可访问：

```text
http://localhost:8888
```

局域网其他电脑访问时，需要访问 Windows 主机 IP 或 WSL 暴露出来的 IP。

查看 Windows IP：

```powershell
ipconfig
```

如果其他电脑访问不通，需要检查：

- Windows 防火墙是否放行 8888 端口
- Nginx 是否监听 `0.0.0.0:8888`
- WSL 网络是否允许局域网访问
- 是否需要 Windows 端口转发

Windows 端口转发示例，以管理员 PowerShell 执行：

```powershell
netsh interface portproxy add v4tov4 listenaddress=0.0.0.0 listenport=8888 connectaddress=172.30.0.50 connectport=8888
New-NetFirewallRule -DisplayName "TaskManageSystem 8888" -Direction Inbound -Protocol TCP -LocalPort 8888 -Action Allow
```

查看转发：

```powershell
netsh interface portproxy show all
```

删除转发：

```powershell
netsh interface portproxy delete v4tov4 listenaddress=0.0.0.0 listenport=8888
```

## 17. 常见故障

### 17.1 502 Bad Gateway

一般是后端没有启动或 Nginx 代理端口不对。

检查：

```bash
pm2 status
curl http://127.0.0.1:4000/health
sudo nginx -t
```

### 17.2 404 Not Found

如果 `/health` 返回 404，通常是 Nginx 没有把 `/health` 代理到后端。

检查 Nginx 配置中的：

```nginx
location /health {
    proxy_pass http://127.0.0.1:4000/health;
}
```

### 17.3 钉钉登录回调跳到 localhost

说明 `.env` 或数据源实例中的回调地址仍是本地地址。

检查：

```bash
cat /home/lizb/TimeManageSystem/.env
```

确认：

```env
PUBLIC_BASE_URL=http://172.30.0.50:8888
FRONTEND_BASE_URL=http://172.30.0.50:8888
```

修改后重启：

```bash
pm2 restart task-manage-api --update-env
```

### 17.4 redirect_uri 参数错误

钉钉开放平台中没有配置当前回调地址。

需要配置：

```text
http://172.30.0.50:8888/api/auth/oauth/dingtalk/callback
```

### 17.5 其他电脑访问不了

检查：

- 是否在同一局域网
- Windows 防火墙是否放行
- 端口是否是 8888
- 是否访问的是正确的 Windows 主机 IP 或服务器 IP
- WSL 是否做了端口转发

### 17.6 项目模块数据未同步或同步较慢

当前项目模块优先读取本地 SQLite 的 `module_rows` 表。新增、编辑、删除会先写入本地，定时同步或手动同步时再推送到钉钉，并从钉钉拉取最新数据回本地。

可检查：

- 系统配置页的同步任务是否成功
- `sync_jobs` 中是否存在失败任务
- `module_rows` 中是否存在 `sync_status = 'failed'` 的行
- 模块配置是否绑定正确数据源和 sheetId
- 服务器访问钉钉开放平台网络是否稳定

## 18. 发布检查清单

每次部署前建议确认：

- 代码已同步到服务器
- `.env` 没有被覆盖
- `server/server-data.db` 没有被覆盖
- `npm run build` 成功
- PM2 状态为 online
- `/health` 返回 `{"ok":true}`
- 钉钉登录回调地址与部署地址一致
- 项目模块能读取数据
- 汇总看板能读取数据
- 消息推送可正常发送
