# 任务管理系统使用文档

## 1. 系统说明

任务管理系统用于管理项目任务、研发进度、提测计划、上线时间、人员分配和消息推送。

系统以本地 SQLite 作为页面查询和编辑的优先数据源，同时按配置与钉钉在线表格或飞书电子表格同步。本地数据库保存账号、权限、数据源配置、模块配置、业务行数据、同步状态、操作日志和消息推送配置。

## 2. 访问地址

生产环境访问地址：

```text
http://172.30.0.50:8888
```

本地开发访问地址：

```text
http://localhost:5173
```

## 3. 登录方式

登录页支持以下方式：

- 钉钉登录
- 飞书登录
- 账号密码登录

钉钉或飞书登录后，系统会根据第三方账号自动创建本地用户。首次进入系统的用户默认是 `viewer` 角色，需要管理员在系统配置中授权后才能获得更多菜单或操作权限。

账号密码登录用于管理员备用登录。普通用户建议使用企业登录。

## 4. 角色说明

系统内置角色：

| 角色 | 说明 |
| --- | --- |
| admin | 管理员，可查看全部数据，维护系统配置、权限、用户、消息推送等 |
| editor | 编辑者，可按授权范围查看、新增、编辑项目数据 |
| viewer | 只读用户，只能查看授权范围内的数据 |

项目数据访问规则：

- 管理员可以查看当前数据源实例下的全部项目数据。
- 非管理员只能看到 `研发人员` 字段包含自己显示名称的数据。
- 非管理员新增项目时，研发人员默认写入当前用户显示名称。
- 已完成的数据不能编辑和删除。

## 5. 首页与菜单

左侧菜单包括：

- 汇总看板
- 项目模块
- 人员信息
- 待办事项
- 系统配置

不同用户看到的菜单取决于权限配置。没有权限的菜单不会显示。

## 6. 汇总看板

汇总看板用于查看当前项目整体进度。

主要内容：

- 进行中总数
- 开发中数量
- 测试中数量
- 总需求数
- 已完成数量
- 整体完成率
- 各模块进度
- 开发中明细
- 测试中明细
- 研发人员任务进展

管理员看到所有项目数据。非管理员只统计研发人员为自己的项目数据。

右上角的“推送到钉钉”按钮用于将当前汇总推送到钉钉群。管理员推送全部汇总，非管理员推送个人汇总。

## 7. 项目模块

项目模块把原来的多个子模块合并到一个页面，通过顶部下拉框切换具体模块。

常见模块：

- 发电标准版
- 售电标准版
- 爬虫
- 分省系统
- 自定义新增模块

项目模块支持：

- 查询
- 按内容过滤
- 按是否完成过滤
- 按计划提测时间过滤
- 按实际提测时间过滤
- 按上线时间过滤
- 按研发人员过滤
- 按产品人员过滤
- 按测试人员过滤
- 排序
- 新增
- 编辑
- 删除

新增和编辑时，公式字段、隐藏字段和序号字段不会显示在表单中。保存后系统先写入本地数据库并标记为待同步，因此当前页面会立即看到最新数据；之后由定时同步或手动同步推送到钉钉，并重新拉取钉钉数据回本地。

## 8. 人员信息

人员信息用于维护各模块中可选择的人员。

当前人员类型包括：

- 产品人员
- 测试人员
- 开发人员

项目模块新增或编辑时，人员字段会从这里读取下拉选项。

## 9. 待办事项

待办事项用于记录临时任务、跟进事项和处理状态。

是否可新增、编辑、删除取决于当前用户的模块权限。

## 10. 系统配置

系统配置主要供管理员使用，包含：

- 数据源实例
- 模块配置
- 用户管理
- 权限管理
- 消息推送

### 10.1 数据源实例

数据源实例用于配置钉钉或飞书表格连接信息。

实例名称是必填项，用于区分不同业务表格。

钉钉常见配置：

| 配置项 | 说明 |
| --- | --- |
| AppKey / Client ID | 钉钉开放平台企业内部应用的 Client ID |
| AppSecret / Client Secret | 应用密钥 |
| CorpId | 企业 ID，部分接口需要 |
| WorkbookId | 钉钉在线表格工作簿 ID |
| OAuth 回调地址 | 企业登录回调地址，不填时使用后端默认地址 |
| 启用企业登录 | 是否允许该实例使用钉钉登录 |

操作者 ID 通常可从企业登录身份或系统配置中自动处理，不建议普通用户手动维护。

### 10.2 模块配置

模块配置用于维护项目模块。

可配置内容：

- 模块名称
- 模块 key
- 分类
- 绑定数据源实例
- 工作表名称
- 是否启用
- 排序

新增模块后，需要点击同步按钮，将模块结构同步到外部数据源。

### 10.3 字段配置

字段配置用于定义模块表头和表单字段。

字段类型包括：

- text
- number
- date
- link
- status
- staff
- formula
- hidden

字段可配置是否隐藏、是否公式字段、是否人员字段、字段顺序等。

### 10.4 用户管理

用户管理用于查看通过钉钉或飞书登录进来的企业成员。

管理员可以：

- 新增可使用账号密码登录的本地用户
- 同步企业成员
- 给已有用户重置账号密码登录密码
- 修改显示名称
- 修改角色
- 启用或停用用户
- 设置默认数据源

注意：显示名称会用于匹配项目表格中的 `研发人员` 字段。若显示名称和表格中的研发人员不一致，非管理员可能看不到自己的项目数据。

账号密码登录入口由数据源实例中的“启用备用登录”或环境变量 `LOCAL_LOGIN_ENABLED=true` 控制。新增本地用户后，如果登录页没有出现账号密码登录入口，需要先开启备用登录。

系统至少需要保留一个启用的管理员账号。若数据库异常导致没有任何启用的管理员，服务启动时会自动恢复默认管理员 `admin/admin123`，登录后请立即修改密码。

### 10.5 权限管理

权限管理用于控制角色或单个用户对模块的访问和操作。

权限包括：

- 查看
- 新增
- 编辑
- 删除

可按角色授权，也可对单个用户单独授权。

### 10.6 消息推送

消息推送用于通过钉钉自定义机器人推送开发中和测试中的任务汇总。

管理员可配置：

- 钉钉机器人 Webhook
- 加签 Secret
- 安全关键词
- 是否启用定时推送
- 每日推送时间

普通用户如有推送权限，可配置个人消息推送和个人定时推送。个人推送只推送自己的项目数据。

### 10.7 数据同步

系统配置页可以查看钉钉 API 使用情况、同步任务、企业成员同步记录和本地数据同步状态。

同步规则：

- 新增、编辑、删除项目数据时先写入本地数据库，记录为待同步。
- 定时同步或手动同步时，先把本地待同步数据推送到对应数据源的钉钉表格，再从钉钉拉取数据回本地。
- 企业成员同步会把钉钉企业成员同步到本地用户，并保留同步日志。
- 同步失败的数据会保留错误信息，下次同步会继续尝试。

## 11. 常见问题

### 11.1 登录后看不到系统配置

当前账号没有系统配置权限。需要管理员登录后，在用户管理或权限管理中授权。

### 11.2 钉钉登录提示 redirect_uri 错误

需要在钉钉开放平台的安全设置中配置正确回调地址。

示例：

```text
http://172.30.0.50:8888/api/auth/oauth/dingtalk/callback
```

### 11.3 登录后只能看到自己的数据

这是正常规则。非管理员只能查看研发人员字段包含自己显示名称的数据。

### 11.4 新增或编辑后没有立即同步到钉钉

这是当前设计。系统保存后先写入本地数据库，页面立即显示；推送到钉钉由系统配置中的定时同步或管理员手动同步触发。可以在系统配置页查看本地数据是否已同步到钉钉。

### 11.5 同步企业成员后角色变回 viewer

正常情况下同步企业成员不会重置已授权角色。如出现异常，需要检查当前部署版本是否为最新版本。

## 12. 使用建议

- 企业成员首次登录后，管理员先进入用户管理确认显示名称和角色。
- 表格中的研发人员名称应与用户管理中的显示名称保持一致。
- 新增模块后先配置字段，再执行同步。
- 钉钉机器人安全关键词建议配置为“项目提醒”“项目名称”“状态”“计划”“链接”等，并在系统消息推送配置中同步维护。
- 数据库表结构见：[数据库表结构](数据库表结构.md)。
