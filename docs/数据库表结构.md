# 数据库表结构

本文档记录任务管理系统当前 SQLite 数据库表、字段结构和主要用途。表结构以 `server/src/db.ts` 中的 `initDatabase()` 为准。

## 1. 数据库概览

- 数据库类型：SQLite，当前项目通过 `sql.js` 读写本地数据库文件。
- 默认数据库路径：`server/server-data.db`。
- 生产部署常用路径：`/home/lizb/TimeManageSystem/server/server-data.db`。
- 可通过环境变量 `DB_PATH` 指定数据库文件路径。
- SQLite `CURRENT_TIMESTAMP` 存储为 UTC 时间；页面展示时按北京时间格式化。

当前架构是“本地数据库优先 + 定时同步外部表格”：

- 页面查询优先读取本地 `module_rows`。
- 新增、编辑、删除先写入本地 `module_rows`，并标记同步状态。
- 定时同步或手动同步时，先把本地待同步数据推送到钉钉，再从钉钉拉取最新数据回本地。
- 企业成员同步会把钉钉企业成员写入本地用户和身份表。

## 2. 表清单

| 表名 | 主要用途 |
| --- | --- |
| `users` | 本地用户、账号密码、角色、启停状态 |
| `user_identities` | 钉钉/飞书等第三方身份与本地用户绑定关系 |
| `data_source_instances` | 数据源实例配置，如不同钉钉表格实例 |
| `module_configs` | 项目模块配置，定义模块 key、名称、绑定数据源和工作表 |
| `module_fields` | 模块字段配置，定义表头、字段类型、人员字段等 |
| `user_data_source_preferences` | 用户当前默认使用的数据源实例 |
| `data_source_staff_assignments` | 当前数据源下的本地人员分组 |
| `module_permissions` | 角色或用户对模块的查看、新增、编辑、删除权限 |
| `audit_logs` | 用户操作审计日志 |
| `notification_settings` | 全局消息推送配置 |
| `notification_logs` | 消息推送执行日志 |
| `notification_user_settings` | 用户个人消息推送配置 |
| `sheet_cache` | 外部表格读取缓存 |
| `api_call_logs` | 外部 API 调用统计与失败记录 |
| `module_rows` | 本地业务数据行，也是页面查询的主要来源 |
| `sync_jobs` | 钉钉推送/拉取同步任务记录 |
| `enterprise_member_sync_logs` | 企业成员同步日志 |
| `system_settings` | 系统级配置，如钉钉定时同步时间 |

## 3. 核心关系

| 关系 | 说明 |
| --- | --- |
| `users.id` -> `user_identities.user_id` | 一个本地用户可绑定一个或多个第三方身份 |
| `data_source_instances.id` -> `module_configs.data_source_id` | 模块绑定到指定数据源实例 |
| `module_configs.module_key` -> `module_fields.module_key` | 每个模块拥有独立字段配置 |
| `data_source_instances.id` + `module_configs.module_key` -> `module_rows` | 本地业务行按数据源和模块隔离 |
| `data_source_instances.id` -> `data_source_staff_assignments.data_source_id` | 不同数据源可维护不同人员分组 |
| `data_source_instances.id` -> `sync_jobs.data_source_id` | 同步任务记录对应数据源 |
| `data_source_instances.id` -> `enterprise_member_sync_logs.data_source_id` | 企业成员同步日志对应数据源 |

当前数据库没有显式声明外键约束，关联关系由服务端代码保证。

## 4. 详细表结构

### 4.1 `users`

本地用户表。账号密码登录、企业登录自动创建用户、角色授权都会使用该表。

| 字段 | 类型 | 约束/默认值 | 说明 |
| --- | --- | --- | --- |
| `id` | INTEGER | PRIMARY KEY AUTOINCREMENT | 用户 ID |
| `username` | TEXT | UNIQUE NOT NULL | 登录账号；企业登录用户通常自动生成 |
| `password_hash` | TEXT | 允许为空 | 账号密码登录的密码哈希 |
| `role` | TEXT | NOT NULL | 角色：`admin`、`editor`、`viewer` |
| `display_name` | TEXT | NOT NULL | 页面展示名称，也是项目人员匹配的重要字段 |
| `enabled` | INTEGER | NOT NULL DEFAULT 1 | 是否启用，1 启用，0 停用 |
| `created_at` | TEXT | NOT NULL DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| `updated_at` | TEXT | NOT NULL DEFAULT CURRENT_TIMESTAMP | 更新时间 |

### 4.2 `user_identities`

第三方身份绑定表。钉钉/飞书登录后，会把第三方用户 ID 与本地用户绑定。

| 字段 | 类型 | 约束/默认值 | 说明 |
| --- | --- | --- | --- |
| `id` | INTEGER | PRIMARY KEY AUTOINCREMENT | 身份记录 ID |
| `user_id` | INTEGER | NOT NULL | 本地用户 ID |
| `provider` | TEXT | NOT NULL | 身份来源，如 `dingtalk`、`feishu` |
| `provider_user_id` | TEXT | NOT NULL | 第三方平台用户唯一标识 |
| `union_id` | TEXT | 允许为空 | 第三方 unionId |
| `open_id` | TEXT | 允许为空 | 第三方 openId |
| `name` | TEXT | 允许为空 | 第三方返回的用户名称 |
| `avatar` | TEXT | 允许为空 | 头像地址 |
| `mobile` | TEXT | 允许为空 | 手机号 |
| `email` | TEXT | 允许为空 | 邮箱 |
| `raw_json` | TEXT | 允许为空 | 第三方原始返回 JSON |
| `created_at` | TEXT | NOT NULL DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| `updated_at` | TEXT | NOT NULL DEFAULT CURRENT_TIMESTAMP | 更新时间 |

唯一约束：`UNIQUE(provider, provider_user_id)`，避免同一第三方身份重复绑定。

### 4.3 `data_source_instances`

数据源实例表。每个实例代表一个钉钉或飞书配置，可对应不同企业应用、工作簿或表格。

| 字段 | 类型 | 约束/默认值 | 说明 |
| --- | --- | --- | --- |
| `id` | INTEGER | PRIMARY KEY AUTOINCREMENT | 数据源实例 ID |
| `name` | TEXT | NOT NULL | 实例名称 |
| `platform` | TEXT | NOT NULL | 平台类型，如 `dingtalk`、`feishu` |
| `config_json` | TEXT | NOT NULL | 数据源配置 JSON，包含 AppKey、AppSecret、WorkbookId、回调地址等 |
| `enabled` | INTEGER | NOT NULL DEFAULT 1 | 是否启用 |
| `sort_order` | INTEGER | NOT NULL DEFAULT 0 | 排序值 |
| `created_at` | TEXT | NOT NULL DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| `updated_at` | TEXT | NOT NULL DEFAULT CURRENT_TIMESTAMP | 更新时间 |

### 4.4 `module_configs`

模块配置表。项目模块、人员模块、待办模块等都通过该表配置。

| 字段 | 类型 | 约束/默认值 | 说明 |

### 4.15 `module_rows`

本地业务数据行表。页面查询项目模块时主要读取该表。

| 字段 | 类型 | 约束/默认值 | 说明 |
| --- | --- | --- | --- |
| `id` | INTEGER | PRIMARY KEY AUTOINCREMENT | 本地记录 ID |
| `platform` | TEXT | NOT NULL | 平台类型 |
| `data_source_id` | INTEGER | NOT NULL | 数据源实例 ID |
| `module_key` | TEXT | NOT NULL | 模块 key |
| `row_id` | TEXT | NOT NULL | 业务行 ID，通常对应外部表格行号 |
| `row_number` | INTEGER | 允许为空 | 外部表格行号 |
| `payload_json` | TEXT | NOT NULL | 业务字段 JSON |
| `sync_status` | TEXT | NOT NULL DEFAULT `synced` | 同步状态：`synced`、`pending`、`failed` |
| `sync_action` | TEXT | NOT NULL DEFAULT `upsert` | 同步动作：`upsert` 或 `delete` |
| `sync_error` | TEXT | 允许为空 | 最近一次同步错误 |
| `synced_at` | TEXT | 允许为空 | 最近成功同步时间 |
| `created_at` | TEXT | NOT NULL DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| `updated_at` | TEXT | NOT NULL DEFAULT CURRENT_TIMESTAMP | 更新时间 |

唯一约束：`UNIQUE(data_source_id, module_key, row_id)`。

同步规则：

- 新增或编辑：写入 `payload_json`，设置 `sync_status = pending`、`sync_action = upsert`。
- 删除：设置或处理为 `sync_action = delete`，页面查询时会隐藏待删除行。
- 同步成功：设置 `sync_status = synced`，清空错误并更新 `synced_at`。
- 同步失败：设置 `sync_status = failed`，记录 `sync_error`，下次同步会继续处理。

### 4.16 `sync_jobs`

同步任务表。记录本地推送到钉钉、钉钉拉取到本地的任务执行情况。

| 字段 | 类型 | 约束/默认值 | 说明 |
| --- | --- | --- | --- |
| `id` | INTEGER | PRIMARY KEY AUTOINCREMENT | 任务 ID |
| `platform` | TEXT | NOT NULL | 平台类型 |
| `data_source_id` | INTEGER | 允许为空 | 数据源实例 ID |
| `module_key` | TEXT | 允许为空 | 模块 key |
| `direction` | TEXT | NOT NULL | 同步方向：`push` 或 `pull` |
| `status` | TEXT | NOT NULL | 状态：`running`、`success`、`failed` |
| `total_rows` | INTEGER | NOT NULL DEFAULT 0 | 处理行数 |
| `message` | TEXT | 允许为空 | 任务说明或错误信息 |
| `started_at` | TEXT | NOT NULL DEFAULT CURRENT_TIMESTAMP | 开始时间 |
| `finished_at` | TEXT | 允许为空 | 结束时间 |

说明：服务启动时会把超过 30 分钟仍为 `running` 的任务标记为失败，避免页面长期显示运行中。

| `updated` | INTEGER | NOT NULL DEFAULT 0 | 本次更新本地用户数 |
| `message` | TEXT | 允许为空 | 任务说明或错误信息 |
| `created_at` | TEXT | NOT NULL DEFAULT CURRENT_TIMESTAMP | 创建时间 |

### 4.18 `system_settings`

系统配置表。用于保存可在系统配置页面调整的全局配置。

| 字段 | 类型 | 约束/默认值 | 说明 |
| --- | --- | --- | --- |
| `key` | TEXT | PRIMARY KEY | 配置 key |
| `value` | TEXT | NOT NULL | 配置值，按字符串保存 |
| `updated_at` | TEXT | NOT NULL DEFAULT CURRENT_TIMESTAMP | 更新时间 |

当前常见配置项：

| key | 说明 |
| --- | --- |
| `dingtalk.sync.enabled` | 是否启用钉钉定时同步 |
| `dingtalk.sync.scheduledTime` | 钉钉每日同步时间，格式 `HH:mm`，按北京时间判断 |
| `dingtalk.sync.startupSyncEnabled` | 服务启动后是否自动同步 |
| `dingtalk.sync.startupDelayMs` | 启动自动同步延迟，毫秒 |

## 5. 运维查看命令

cd /home/lizb/TimeManageSystem
